## Key Bindings & Navigation

- **`modePluginView` fully surrenders the keyboard â€” argus reserves NOTHING but the double-Ctrl+Q failsafe.** Every key (Esc, Ctrl+C, `?`, tab numbers, modified arrows) returns the event so tview routes it to the focused pane. The ONLY interception is a fast double Ctrl+Q (`pluginFailsafeWindow`, 400ms): the first press is recorded in `a.lastCtrlQ` and forwarded; a second within the window intercepts (returns nil) and force-returns via `deactivatePluginView`. Two presses outside the window do NOT fire.
- **`a.lastCtrlQ` needs no mutex but MUST be reset in `activatePluginView`.** It's touched only on the tview main goroutine (via `SetInputCapture` â†’ `handleGlobalKey`). Without the reset, a stale timestamp from a prior plugin view could trip the failsafe on the very first Ctrl+Q in a new view.
- **The failsafe window reads `a.nowFn` (defaults to `time.Now`).** Tests override it to drive the 400ms window deterministically â€” a real clock makes the double-tap timing flaky.
- **`keyenc` maps `tcell.KeyCtrlQ` â†’ `{0x11}`.** Added for plugin-view surrender so a single Ctrl+Q forwards. Safe for the agent view because `handleAgentKey` intercepts Ctrl+Q before the encoder ever runs.
- **The plugin help overlay (`?` is plugin-triggered, NOT argus-reserved) is the ONE other key argus intercepts in `modePluginView` besides the failsafe.** The plugin sends `{"type":"help"}`; `requestPluginHelp` pops a `modal.NewHelpModalWith(title, pluginHelpSections(mount.hotkeys))` rendering the FULL dictionary (bar flag ignored) on the `pluginHelpPage`. While `a.pluginHelpVisible` is true, the `modePluginView` branch consumes exactly the NEXT key via `dismissPluginHelp` (checked BEFORE the Ctrl+Q failsafe) and returns nil â€” the overlay does not capture the keyboard beyond that single dismissal. `?` itself is never inspected by argus; it forwards to the plugin like any other key.
- **`deactivatePluginView` must `RemovePage(pluginHelpPage)` directly, not call `dismissPluginHelp`.** dismissPluginHelp refocuses `a.activePlugin`, which deactivate has already nil'd â€” so deactivate tears the page down inline and clears `pluginHelpVisible` to stop the overlay bleeding into the next plugin.
- **`ctrl+c` only exits from task list view.** In agent mode, writes `0x03` to PTY (or no-op if dead).
- **`ctrl+q` in diff mode must exit diff AND refocus terminal.** Otherwise user needs a second keypress.
- **`ctrl+d` exits agent view when session is dead.** Without this, Ctrl+D after agent exit is silently dropped.
- **`ctrl+l` opens fuzzy link picker (works while agent runs).** Uses `tcell.KeyCtrlL` â€” reliable across all terminals (unlike `ctrl+/` which has inconsistent encodings). Overrides the typical "clear screen" behavior; the agent PTY never sees it. Reads session log in a background goroutine; the `QueueUpdateDraw` callback must guard `a.mode == modeAgent` because the user may leave agent view during I/O.
- **`ctrl+p` shells out to `gh pr view --web` in the current worktree.** No PR URL is tracked locally â€” gh discovers the PR from the branch + remote. Intercepted before PTY pass-through; the agent never sees `0x10`. No-op when `worktreeDir` is empty. `prOpener` is a package-level seam so tests stub the exec. From the task list (`modeTaskList` on the Tasks tab), uses the selected task's `Worktree` directly â€” no-op for pending tasks with no worktree yet.
- **Escape in agent view:** Refocuses terminal from diff/files but does NOT exit agent view. Always returns `nil` to consume the event.
- **Mouse clicks must update `agentFocus`, not just tview focus.** Custom `MouseHandler` overrides needed.
- **In diff mode: Up/Down switch files, j/k scroll diff.**
- **Cmd+Up/Down navigates between tasks in agent view** via `ModCtrl|ModAlt` check.
- **tcell has no `KeyCtrlLeft`/`KeyCtrlRight`.** Check `event.Modifiers()&(tcell.ModCtrl|tcell.ModAlt) != 0`.
- **`exitAgentView` must reset `header.SetTab(TabTasks)` and `statusbar.SetTab(TabTasks)`.** The global key handler routes non-agent keys based on `header.ActiveTab()` â€” if the tab isn't reset, up/down keys get routed to the wrong tab's handler (e.g., Settings) instead of the task list.
- **Settings `d` key is context-dependent: project rows â†’ delete (with confirmation), backend rows â†’ set default.** `handleDeleteOrDefault` dispatches on `currentSection()`. Deleting a project orphans its tasks (no FK constraint) â€” the confirmation modal counts tasks via `a.tasks` and warns the user.
- **`ctrl+y` in agent view is conditionally intercepted â€” only steals from the PTY when an agent has staged a clipboard payload.** With nothing staged, `copyStagedClipboard()` returns false and the switch falls through to the PTY pass-through path so vim/emacs `yank-pop` still reaches the underlying program. The intercept fires synchronously on the tview goroutine; the actual pbcopy and clear-RPC run in goroutines.
- **`ctrl+z` in agent view toggles single-pane (zoom) and MUST be intercepted before the PTY.** `tcellKeyToBytes` maps `KeyCtrlZ` to `0x1a` (SIGTSTP); if it reached the PTY, Claude Code backgrounds the foreground task. `handleAgentKey`'s `KeyCtrlZ` case calls `toggleAgentZen()` and returns `nil` so the byte never forwards (same unconditional-steal pattern as `ctrl+l`/`ctrl+p`).
- **Zen mode collapses panes via `agentPanels.ResizeItem(item, 0, 0)`, not by removing them from the flex.** Restore is `ResizeItem(item, 0, 1)`. Don't add a `screen.Sync()` â€” the terminal pane's `Draw()` recomputes PTY size from its own inner rect every frame, so the resize RPC + agent full-width repaint fire automatically (this is a real resize, not a ghost-cell artifact). Toggling does NOT touch `computePTYSize` (that path only seeds the initial size on agent-view *entry*, where zen is always reset to false).
- **`toggleAgentZen` forces `agentFocus` back to `focusTerminal` when zooming.** The file panel is hidden at width 0; leaving focus there would route keys to an invisible widget. Un-zoom keeps terminal focus (no prior-focus tracking). `clearAgentZen()` (the idempotent reset shared by un-zoom, `exitAgentView`, and both agent-view entry points) sets `agentZen=false` and restores the 1:3:1 proportions. The defensive call on entry guards against a session torn down without `exitAgentView`. The restored widths only lay out on the next draw of the (then-hidden) agent page, so verify restore by re-entering, not by reading `GetRect` immediately after exit.
