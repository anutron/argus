## Daemon & RPC

- **Reconciliation must distinguish RPC failure from empty running set.** `Client.Running()` returns `nil` on RPC error vs `[]string{}` when daemon confirms no sessions. Check `runningIDs != nil && !daemonRestarting` before reconciling — during restart the new daemon has no sessions, so reconciliation would incorrectly mark all InProgress tasks Complete.
- **All daemon RPC calls must have a timeout.** `net/rpc.Client.Call()` blocks indefinitely. Use the `c.call()` wrapper with `select + time.After`. The buffered channel (`make(chan error, 1)`) is critical to prevent goroutine leak.
- **Stream loss ≠ process exit.** Use `StreamLost` flag to distinguish. When `isSessionAlive()` returns `reachable=false`, treat as stream lost, not exit. **The `<-rs.done` path in `connectStream`'s retry loop must also use `removeSessionStreamLost`** — `Client.Close()` during daemon restart closes `done`, and using `removeSession` causes a zero-value ExitInfo (StreamLost=false) because the GetExitInfo RPC fails on the closed client, incorrectly marking the task Complete.
- **`client.Get()` must return `nil` for any non-alive session.** Check `!info.Alive` alone — PID is stale between `onFinish()` and `delete(r.sessions)`.
- **Daemon health check uses `Ping()`, not `refreshTasks()`.** Ping fails fast; refreshTasks blocks on RPC.
- **Never call `refreshTasks()` from the tview main goroutine.** It blocks on RPC. Use `refreshTasksAsync()` (spawns goroutine + QueueUpdateDraw) or `refreshTasksLocal()` (reuses cached IDs, no RPC). Use `refreshTasksLocal` when only DB state changed (delete/prune); use `refreshTasksAsync` when session state may have changed.
- **Never call `refreshTasksAsync()` between `db.Add(task)` and `startSession()`.** The async RPC snapshot captures running IDs before the session exists, then reconciliation sees InProgress + not-in-running-set → marks Complete. Use `refreshTasksLocal()` instead (no RPC, no reconciliation race).
- **`refreshTasksAsync` (and any other async RPC->reconciliation path) must capture `startGen` before the RPC and check it in the callback.** Without this guard, a session exit calling `refreshTasksAsync` while another task is starting captures stale runningIDs that don't include the new session — reconciliation marks it Complete. Same guard pattern as `onTick`.
- **Task creation must call `SelectByID` before `onTaskSelect`.** The ordering is `refreshTasksLocal()` → `tasklist.SelectByID(id)` → `onTaskSelect(task)`. Without `SelectByID`, the cursor stays on the previous row — when the user exits agent view, the task list highlights the wrong task. `SelectByID` must come before `onTaskSelect` because agent view covers the task list; the cursor position matters on return.
- **`startSession` must bump `startGen` before `runner.Start()`.** The tick goroutine captures `runningIDs` via RPC independently — if it snapshots before the session exists, the stale IDs reach reconciliation (which runs on the tview goroutine AFTER startSession). `onTick` captures `startGen` before the RPC; the `QueueUpdateDraw` callback checks if it changed and passes `nil` runningIDs to skip reconciliation for that tick. Next tick gets fresh IDs.
- **Always use `RunningAndIdle()` instead of separate `Running()` + `Idle()`.** They share the same `ListSessions` RPC — calling both separately doubles RPC overhead and doubles head-of-line blocking risk on the single `net/rpc` connection.
- **Daemon cleanup must run on the `Serve` goroutine, not `Shutdown`.** `Shutdown()` runs on a non-main goroutine — process may exit before cleanup completes. `signal.Stop(sigCh)` must be called after the handler goroutine exits.
- **Daemon restart must preserve `SessionID` on tasks.** Clearing it loses conversation history. On re-launch, `--resume <id>` picks up where it left off.
- **Rebuilding the binary does NOT update the running daemon.** Kill and restart: `kill -TERM $(cat ~/.argus/daemon.pid)`.
- **`onFinish` must fire before session removal.** Callback runs OUTSIDE `r.mu` (to avoid deadlock), then `delete(r.sessions)` in a second lock section.
- **`startOrAttach` must revert all state on Start failure.** Reset to Pending, clear SessionID, zero StartedAt. `SetStatus(StatusPending)` doesn't clear StartedAt — explicit zero required.
- **Daemon session exits must be wired to TUI via `client.OnSessionExit`.** Without this, tasks stay InProgress forever in daemon mode.
- **`restartDaemon` must re-wire `OnSessionExit` on the new client.**
- **Fresh daemon auto-start AND daemon restart must set `daemonFreshStart=true`.** Reconciles stale InProgress tasks to InReview (not Complete) on the first tick. Without this on the restart path, tasks that lost their sessions during the restart cascade get incorrectly marked Complete. The flag is cleared after the first reconciliation so subsequent ticks use normal Complete behavior.
- **Daemon restart cascade: health check must have a 30s cooldown between restart attempts.** Without the cooldown, each auto-resume failure triggers another restart, creating a feedback loop (13 restarts in 40 seconds observed). The cascade kills sessions that were just started, causing orphaned processes and stale `connectStream` goroutines. **Both restart paths must set `lastDaemonRestart`** — the health check path AND the manual Settings `OnRestartDaemon` callback. Without setting it on manual restart, the health check can trigger another restart immediately (no cooldown).
- **`onTick` must read `a.runner` under `a.mu`.** The tick goroutine reads `a.runner` for `RunningAndIdle()`. During a restart cascade, `restartDaemon`'s `QueueUpdateDraw` writes a new client to `a.runner` on the tview goroutine. Without synchronization, the tick can use a stale client whose `ListSessions` returns empty → reconciliation marks all InProgress tasks Complete.
- **`Client.Close()` must signal `closed` channel before closing sessions.** `connectStream` goroutines check `c.closed` at the top of each retry iteration. Without this, old goroutines from a replaced client keep connecting to the new daemon's socket, flooding it with "session not found" requests.
- **Reconciliation must skip tasks started within the last 5 seconds (`recentStarts` grace period).** Belt-and-suspenders alongside `startGen`. After a restart cascade, `ListSessions` can return stale data if the RPC goes to an intermediate daemon. The grace period protects recently-started tasks from false reconciliation.
- **SessionID must be populated before first `runner.Start` for Claude backends, and captured post-exit for Codex.** Claude uses `--session-id <uuid>` on first run and `--resume <uuid>` on subsequent runs. Codex captures its ID from `~/.codex/state_5.sqlite` after exit. Without a SessionID, `resume` is always false and every start is a fresh conversation.
- **Codex session ID capture (`CaptureCodexSessionID`) must run off the tview main goroutine.** It opens a SQLite connection which can block. Use a background goroutine + `QueueUpdateDraw` to persist.
- **`onTaskSelect(task, autoStart)` auto-starts sessions when `autoStart=true` for any non-complete, non-archived task without a running session.** No explicit `startSession` call is needed after `onTaskSelect` — adding one causes a double-start. Pass `autoStart=false` for navigation-only entry (e.g., `navigateAgentTask`). The "user left pending view" path in `handleNewTaskKey` is the one exception (calls `startSession` without `onTaskSelect`). Condition excludes completed and archived tasks.
